3. broker/tof_broker.py (43 lines)
   - TOFBroker class: boulder/pebble classification
   - compute_ec(): Execution Cost = MI / FOG_MIPS
   - classify(): returns "boulder" if EC ≥ 1.0s else "pebble"
   - process_dag(): processes full DAG and collects statistics
   - Statistics tracking for analysis

4. optimizer/nsga2_mmde.py (230 lines)
   - TaskOffloadingProblem: NSGA-II problem formulation
     - Objectives: minimize energy, minimize latency
     - Variables: routing decisions for pebble tasks
   - MMDEMutation: Differential Evolution operator
     - Replaces standard mutation with directional mutation
     - V = r1 + F * (r2 - r3)
   - run_nsga2_mmde(): generates Pareto front solutions
   - extract_training_pairs(): converts solutions to (state, action) pairs
   - build_state_from_step(): creates 13-dim state vectors

5. agents/dqn.py (47 lines)
   - DQNNetwork: configurable multilayer perceptron
     - Shared architecture for both agents
     - Input: state dimensions, Output: action Q-values
   - ReplayBuffer: experience replay for training
     - Push: store (state, action, reward, next_state, done)
     - Sample: random batch for mini-batch training

6. agents/agent1.py (102 lines)
   - Agent1: Task placement DQN
   - select_action(): epsilon-greedy action selection
   - pretrain(): behavioral cloning from NSGA-II
   - store(): store transition in replay buffer
   - update(): DQN training step with target network
   - compute_reward(): R = -0.5*latency - 0.3*energy - 0.2*violations
   - save/load: checkpoint persistence

7. agents/agent2.py (97 lines)
   - Agent2: SDN routing DQN
   - preinstall_vip_lane(): proactive OpenFlow rule installation
   - route_flow(): reactive vs proactive routing decision
   - select_action() & update(): equivalent to Agent1
   - compute_reward(): delivery + delay + overhead trade-off
   - Action space: PRIMARY, ALT1, ALT2, VIP_RESERVE, BEST_EFFORT

8. mobility/handoff.py (99 lines)
   - TrajectoryPredictor class:
     - compute_distance(): Euclidean distance
     - compute_t_exit(): vehicles leave fog coverage time
     - compute_t_exec(): task execution time with load
     - predict_next_fog(): next fog destination prediction
     - select_mode(): DIRECT vs PROACTIVE handoff
   - HTBBuffer: Handoff Task Buffer for disconnected vehicles
     - push(): task moves from NTB to HTB
     - deliver_on_reconnect(): results delivered when vehicle reconnects

9. results/metrics.py (34 lines)
